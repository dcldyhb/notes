# Prompt 工程

## 一、什么是 Prompt 工程

**Prompt Engineering（提示工程）**是指：

> 通过设计和优化输入给大语言模型（LLM）的提示（Prompt），从而获得更准确、更稳定输出的一种技术。

在使用大语言模型（如 GPT、Claude 等）时，模型输出不仅取决于模型本身能力，还取决于 **Prompt 的设计质量**。

可以简单理解为：

```
模型效果 ≈ 模型能力 × Prompt 质量
```

Prompt 工程的核心目标：

- 明确任务
- 提供必要上下文
- 规范输出结构
- 引导模型推理

本质上可以理解为：

> **用自然语言设计一种可控的“AI程序接口”。**

---

## Prompt 的基本结构

一个高质量 Prompt 通常包含四个部分：

| 组成部分                  | 作用                     |
| ------------------------- | ------------------------ |
| Role（角色）              | 设定 AI 的身份或专业领域 |
| Task（任务）              | 明确需要完成的任务       |
| Context（上下文）         | 提供相关信息             |
| Output Format（输出格式） | 限制输出结构             |

示例：

```
角色：
你是一名资深 Python 工程师。

任务：
解释下面代码的作用。

代码：
[代码]

输出要求：
1. 解释代码逻辑
2. 指出潜在问题
3. 给出优化建议
```

这种结构化 Prompt 可以显著提高回答质量。

---

## 如何写好 Prompt

### 1 明确任务

Prompt 应该具备清晰目标。

不好的 Prompt：

```
帮我看看这个
```

好的 Prompt：

```
总结以下文章的核心观点，并用3条 bullet point 表示。
```

---

### 2 提供上下文

模型需要上下文进行推理。

不好的 Prompt：

```
解释这个函数
```

好的 Prompt：

```
以下是一个 Flask API：

[代码]

请解释该 API 的工作流程和用途。
```

---

### 3 指定输出格式

明确输出格式可以减少不稳定性。

例如：

```
请按 JSON 格式输出：

{
 "title": "",
 "summary": "",
 "keywords": []
}
```

或者：

```
用表格形式输出结果
```

---

### 4 设定角色

角色可以控制回答的专业程度。

示例：

```
你是一名数据科学家
```

```
你是一名经验丰富的软件工程师
```

```
你是一名计算机科学教授
```

---

### 5 限制输出范围

为了避免回答过于发散，可以增加限制。

例如：

```
总结以下文章：

要求：
- 不超过150字
- 用通俗语言
```

## 常见 Prompt 技术

### 1 Zero-shot Prompt

不提供示例，直接给任务。

```
将以下中文翻译成英文：

我今天很开心。
```

特点：

- 简单
- 适用于基础任务

---

### 2 Few-shot Prompt

提供示例让模型模仿。

```
例子：

中文：你好
英文：Hello

中文：谢谢
英文：Thank you

现在翻译：

中文：我喜欢学习
英文：
```

优点：

- 提高准确率
- 控制输出风格

---

### 3 Chain-of-Thought（CoT）

引导模型逐步推理。

```
一步一步思考：

小明有3个苹果
又买了5个苹果
所以总数 = 3 + 5

答案：
```

适用于：

- 数学问题
- 逻辑推理
- 复杂任务

## Prompt 示例

### 文章总结示例

```
角色：
你是一名专业编辑。

任务：
总结以下文章。

文章：
[文本]

要求：
1. 提取核心观点
2. 用通俗语言
3. 不超过120字

输出格式：
- 核心观点
- 关键结论
```

---

### 代码解释示例

```
角色：
你是一名资深软件工程师。

任务：
解释以下代码。

代码：
[代码]

要求：
1. 解释代码逻辑
2. 指出潜在问题
3. 给出优化建议
```

---

### 产品分析示例

```
角色：
你是一名产品经理。

任务：
分析以下产品 idea。

Idea：
AI 自动总结 YouTube 视频。

输出：
1. 产品价值
2. 目标用户
3. 核心功能
4. 商业模式
```

### 课件提取笔记

```
角色：
你是一名资深 AI 学习笔记整理专家，熟悉 Markdown 格式和结构化笔记规范。

任务：
根据以下课程 PDF 内容，提取核心知识点，并生成完整的 Markdown 笔记。
笔记要求模仿现有笔记格式：

- 一级标题（#）用于课程或章节名称
- 二级标题（##）用于主题或主要知识点
- 三级标题（###）用于子知识点或示例
- 使用代码块、表格或 bullet points 来组织关键内容
- 在适当位置添加示例、公式或提示技巧
- 生成内容必须结构化、清晰，并便于复习

输入：
[在这里放置 PDF 文本内容或转文本后的内容]

输出格式要求：

1. Markdown 格式
2. 保持标题递减层级，不使用编号
3. 包含至少以下部分：
   - 课程章节标题
   - 主题总结
   - 示例（代码或表格）
   - 提示或小技巧（如果适用）
4. 输出直接可用，不需要额外处理

注意：

- 不要遗漏核心概念
- 使用清晰简洁的语言
- 避免重复和冗余内容
- 如有公式，请用 Markdown 支持的 LaTeX 语法
```

## Prompt Cheatsheet

常用 Prompt 速查表：

### 角色设定

```
你是一名[领域专家]。
请以专业、清晰的方式回答问题。
```

### 总结任务

```
总结以下内容：

要求：
- 提取核心观点
- 使用3个 bullet points
- 不超过100字
```

### 结构化输出

```
请按照以下 JSON 格式输出：

{
  "title": "",
  "summary": "",
  "keywords": []
}
```

### 推理问题

```
请一步一步思考，并给出最终答案。
```

### 防止幻觉

```
如果你不确定答案，请直接说明“我不知道”，不要编造信息。
```

### 代码分析

```
解释以下代码：

请包括：
1. 代码功能
2. 执行流程
3. 潜在问题
```

### Prompt 设计公式

```
好 Prompt =
角色 + 任务 + 上下文 + 输出格式 + 限制条件
```

这个 Cheatsheet 可以作为快速设计 Prompt 的参考。

## 总结

Prompt 工程核心原则：

```
好 Prompt =
明确任务
+ 提供上下文
+ 指定输出格式
+ 限制输出范围
```

通过合理设计 Prompt，可以显著提高 AI 输出质量与稳定性。
